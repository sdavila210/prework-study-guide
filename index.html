HTML

  • The head element contains information about the webpage.
  • The body element represents the visual content shown to the user.
  • HTML is used to structures a webpage and provide a posistive user experience.
  • We want to use semantic HTML elements to indicate the role of their content and to give structure to the webpage.
  • An element is the entire element from opening tag to closing tag.
  • A tag is what appears inside the angle brackets
  • ul element represents unordered lists
  • ol element represents ordered lists
  • li element represents list items representated in bullet or number lists
  • img element contains information about images displayed on the webpage.
  • br element creates a line of empty space
  • We will continue to learn and use different elements and tags, such as the header tag, as we build our study guide.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are three ways to style a webpage using CSS
  • Inline CSS: applies to specific HTML tag, using a style attribution with a CSS rule to style a particular page elemtn only.
  • Internal CSS style sheet: contains CSS rules for the webpage in a head sections for the HTML file. These rules only apply to that page.
  • External CSS style sheet: allows developers to keep their CSS rules in a separate file, which makes design changes easier. We link a file using LINK element placesd in the HTML head.
  • Separation of concerns is a term referring to separating code into different files based on use.
  • A selector defines the element or attributes to which the rules or declarations will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.

Git

  • git status: checks what branch we are currently on.
  • git check out -b branch-name: creates a new branch and switches to it.
  • We create feature branches so that we can use version control which is being able to track changes in our code.
  • version control also allows us to collaborate with other developers more easily.
  • We access git through the command line and use the command git at the beginning.
  • Git is great for tracking changes locally on your own computer, but we use GitHub to collaborate with others.
  • Github is cloud-based hosting service that allows us to keep track of our projects in repositories.
  • Repositories, or repos, are folders containing: files that contain code, files that contain information about the code such as README file, and version history for all the files in the repo.

JavaScript

  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
  • String: enclosed sequence of text in in single or double quotation marks.
  • Number: numbers dont have quote around them
  • Boolean: true/false values. The words true and false are special keywords that do not need quotes.
  • Array: a structure that allows you to store multiple values in a single regerence. It stores related data.
  • Index: a location number to accept individual data inside the array, starting at 0.
  • For loops: uses predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • Object: This can be anything. Everything in JavaScript is an object and can be stored in a variable.
  • Comments: snippets of text that can be added along with code. The browser will ignore text marked as comments.
  • Conditionals: code structures used to test if an expression returns true or not. Example: if...else statements.
  • We use the console.log method to log variables to the web console browser.
  • We connect script.js to index.html by adding script element under the footer element in the body.
  • Function: a set of instructions that tells the computer how to perform a certain task.
  • Defining a function does not mean it will execute automatically, we must call it in our code.
