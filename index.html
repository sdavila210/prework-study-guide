HTML

  • The head element contains information about the webpage.
  • The body element represents the visual content shown to the user.
  • HTML is used to structures a webpage and provide a posistive user experience.
  • We want to use semantic HTML elements to indicate the role of their content and to give structure to the webpage.
  • An element is the entire element from opening tag to closing tag.
  • A tag is what appears inside the angle brackets
  • ul element represents unordered lists
  • ol element represents ordered lists
  • li element represents list items representated in bullet or number lists
  • img element contains information about images displayed on the webpage.
  • br element creates a line of empty space
  • We will continue to learn and use different elements and tags, such as the header tag, as we build our study guide.

Git

  • git status: checks what branch we are currently on.
  • git check out -b branch-name: creates a new branch and switches to it.
  • We create feature branches so that we can use version control which is being able to track changes in our code.
  • version control also allows us to collaborate with other developers more easily.
  • We access git through the command line and use the command git at the beginning.
  • Git is great for tracking changes locally on your own computer, but we use GitHub to collaborate with others.
  • Github is cloud-based hosting service that allows us to keep track of our projects in repositories.
  • Repositories, or repos, are folders containing: files that contain code, files that contain information about the code such as README file, and version history for all the files in the repo.
